Capabilities

What MyoNeuroTwin does for you

You bring a DEG table (gene, logFC, pvalue, padj) from human or mouse. MyoNeuroTwin does the rest: hubs, pathways, TFs, drugs, ML and digital twins.

Workflow

From DEG table to digital twin ‚Äì in one click

Underneath the Shiny UI, a coordinated R + Python backend executes a deterministic analysis chain. Each step can also be called headless for scripted batches.

Step 1 ¬∑ Upload

Upload DEG (human or mouse)

Drop a CSV/TSV with gene, logFC, pvalue, padj. The app validates column names, infers species (human vs mouse) and stores a clean DEG object in the Shiny session.

Step 2 ¬∑ Hub scoring

STRING network & hubs

Genes are pushed through STRINGdb to build a PPI subnetwork. Hub genes are scored by MCC, betweenness and |logFC|, then aggregated into a composite HubScore_Sum.

Step 3 ¬∑ Enrich

Pathways & TFs

The ranked hubs feed into Enrichr, querying GO/KEGG/Reactome and TF libraries. TF/regulon summaries highlight upstream transcriptional controllers for your model.

Step 4 ¬∑ Drugs

Drug signature matching

Enrichment outputs are merged with DSigDB/DrugMatrix signatures to identify drugs whose perturbational profiles best oppose the disease expression pattern.

Step 5 ¬∑ ML

Random forest prioritisation

A Random Forest classifier is trained on simulated ‚Äúrescue vs non-rescue‚Äù labels to rank hub genes, pathways and drugs by predictive importance.

Step 6 ¬∑ Twin

Digital twin trajectory

The final layer generates a simple digital twin representation ‚Äì trajectories in a feature space that can be compared across models (e.g. SBMA vs SETX vs SOD1).

Deployment

Open-source first, clinical-grade later

Start as an open Shiny app, then graduate to containerised deployments with a Python backend and job queue for heavier workloads.

Phase 1 ‚Äî Shiny-first deployment

  • Host the R Shiny app on shinyapps.io or Posit Connect.
  • Reticulate attaches a project-local Python virtualenv with the enrichment & ML pipeline.
  • Suitable for lab-internal users (neuromuscular group, collaborators, students).
  • Outputs are written to the server file system for download (ZIP export).

Phase 2 ‚Äî Container & paywalled access

  • Dockerise the Shiny + Python stack with pinned R and Python versions.
  • Expose a clean HTTP API for the pipeline; Shiny/Streamlit front-ends talk to it.
  • Deploy on institutional Kubernetes or low-cost cloud (fly.io, DigitalOcean, etc.).
  • Later: add authentication + subscription tiers for pharma / biotech users.
